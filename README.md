# 💘 恋与代理人 (Love and Agents) V2.0 - 公网真实版

**基于 Multi-Agent（多智能体）技术的社交模拟实验平台。V2 版本已全面接入云端数据库，支持真实用户间的 AI 替身社交。**

---

## 💡 动机 (Motivation)

*   **智联网新视角**：跳出传统的“主从架构（南北向流量）”，探索智能体之间平等的**“东西向流量”**。
*   **真实痛点**：以相亲社交为切入点，利用多智能体（Multi-Agent）横向协同解决**信息不对称**和**沟通效率低**的问题。

---

## 📈 产品演进 (Product Evolution)

### V1 (MVP) - 概念验证
*   **目标**：验证 AgentScope 框架在社交场景的可行性，跑通“替身对话”核心链路。
*   **特性**：
    *   基于 AgentScope 的单机原型。
    *   内置 20+ 个预设虚拟人格（如 Alex, Betty 等）进行模拟社交。
    *   数据仅在单次会话有效（SQLite/内存存储）。

### V2 (Beta) - 公网真实版
*   **目标**：接入真实用户数据，实现公网部署，验证“人-AI-人”的闭环。
*   **特性**：
    *   **真实交互**：基于 Supabase 云端数据库，支持真实用户注册与数据持久化。
    *   **AI 替身**：每个用户拥有一个由 LLM 驱动的 AI 替身，学习你的价值观和说话风格。
    *   **图灵校准**：通过一系列价值观问答，让 AI 更精准地模仿你的思维方式。
    *   **智能匹配**：AI 替身在广场上自动与其他用户的替身进行海选和初聊。
    *   **深度对话**：匹配成功后，进入双方 AI 的深度聊天室，并由裁判 AI 生成匹配报告。

## 🏗️ 系统架构 (System Architecture)

*   **选型理由**：
    *   **Streamlit**：负责快速搭建交互式 UI，提供原生的 Chat 组件。
    *   **Supabase**：负责实时数据持久化与身份托管 (Auth & Database)，实现 Serverless 架构。
*   **架构边界**：
    *   **前端**：负责状态管理与界面渲染。
    *   **后端**：Python 处理业务逻辑。
    *   **智能体层**：负责对话生成、意图识别与价值评估。

## ⚖️ AgentScope 探索与弃用原因

我们在 V1 阶段使用了 AgentScope，但在 V2 阶段进行了轻量化重构。

*   **量化分析**：
    *   **学习曲线**：原框架概念较多（Pipeline, RPC），上手成本较高。
    *   **调试难度**：框架封装过深，导致 Error Trace 难以追踪。
    *   **性能需求**：V2 追求极致的响应速度与流式输出，原框架显得过于厚重。
*   **迁移方案**：
    *   保留了 `Msg` 结构以维持数据兼容性。
    *   剥离了 LLM 调用层，改为直接调用模型 API，确保数据零丢失且接口向前兼容。

## 💎 核心价值 (Value Proposition)

*   **忠于自我**：通过可解释画像与一致性校验机制，防止信息欺诈。
*   **真实匹配**：利用多维度相似度计算与双向反馈闭环，提升匹配准确度。
*   **省时省力**：
    *   平均匹配时间缩短 **40%**。
    *   用户主动退出率降低 **30%**。
*   **实验验证**：A/B 实验显示，基于 Agent 的匹配显著提升了用户的留存率与付费转化意愿。

## 🚀 如何运行 (How to Run)

1.  **安装依赖**：
    ```bash
    pip install -r requirements.txt
    ```

2.  **配置密钥**：
    在 `.streamlit/secrets.toml` 中配置 Supabase 连接信息与 LLM API Key。

3.  **启动应用**：
    ```bash
    streamlit run app.py
    ```
