# 产品需求文档 (PRD) - 恋与代理人 (Love and Agents)

> 版本: v1.0  
> 日期: 2026-02-11  
> 状态: 已确认 (Approved)

## 1. 产品愿景 (Product Vision)

**“恋与代理人”** 是一款基于大语言模型（LLM）的 Agent-to-Agent 代理相亲平台。
核心理念是**“替身社交”**：用户通过配置专属 Agent，由 Agent 代替用户在虚拟广场进行海量社交与初步筛选，最终仅将高质量的匹配对象（及聊天摘要）反馈给用户。
旨在解决现代人社交精力有限、沟通成本高、筛选效率低的问题。

## 2. 核心功能模块 (Core Modules - Phase 1)

### 2.1 专属 Agent 构造 (Agent Construction)
**目标**: 打造一个极其接近用户本人性格与审美的数字替身。

*   **基础属性 (Hard Attributes)**:
    *   **个人信息**: 年龄、身高、体重、职业、年薪范围、居住地。
    *   **硬性择偶标准**: 必须接受的年龄差、身高底线、地域限制等。
*   **性格克隆 (Persona Cloning)**:
    *   **MBTI 设定**: 注入用户的 MBTI 类型（如 INFP, ENTJ），决定 Agent 的思维模式。
    *   **兴趣标签**: 3-5 个核心兴趣（如：科幻电影、马拉松、价值投资）。
    *   **图灵校准 (Turing Calibration)**: 用户需回答 5-10 个预设的情境题（例：“对方约会迟到半小时，你会怎么做？”），Agent 学习用户的语气与决策逻辑。

### 2.2 自动相亲系统 (Auto-Dating Engine)
**目标**: 在无人值守的情况下，高效筛选匹配对象。

*   **第1层：硬筛选 (Hard Filter)**
    *   基于 SQL/Vector 的快速过滤。
    *   排除不符合硬性择偶标准的对象（如距离过远、年龄不符）。
*   **第2层：破冰 (Ice Breaking)**
    *   双方 Agent 进行 3-5 轮短对话。
    *   目的：确认对方活跃度、基本礼貌及语言风格是否排斥。
*   **第3层：深度交流 (Deep Dive)**
    *   针对高潜力对象，开启 20+ 轮的长对话。
    *   **话题卡机制**: 系统随机抛出话题卡（如：“最难忘的一次旅行”、“对未来家庭的规划”），引导 Agent 进行深度价值交换。

### 2.3 评估与反馈系统 (Evaluation & Feedback)
**目标**: 将海量聊天记录转化为用户可读的高价值信息。

*   **AI 情感打分**: 
    *   Agent 内部维护一个评分模型（0-100分）。
    *   打分维度：共同话题密度、情绪响应值、价值观契合度。
*   **每日日报 (Daily Report)**:
    *   **数据概览**: 今日聊了多少人，淘汰了多少人。
    *   **今日最佳 (Top Pick)**: 推荐一位分数最高的嘉宾。
    *   **精彩切片 (Highlights)**: 截取 3 段最有意思的对话片段（如：最幽默的回复、最深度的共鸣）。
*   **每周周报 (Weekly Match)**:
    *   基于一周数据生成的“约会推荐名单” (Top 3)。
    *   提供详细的推荐理由（Agent 的观察报告）。

## 3. 用户交互流程 (User Journey)

1.  **初始化 (Setup)**:
    *   用户下载 App -> 填写基础资料 -> 完成性格校准问答 -> Agent 激活。
2.  **挂机运行 (Idle)**:
    *   用户退出 App，Agent 在云端后台持续运行，进入“虚拟广场”寻找匹配对象。
3.  **结果推送 (Notify)**:
    *   每日晚 8:00 推送日报。
    *   用户查看日报，浏览精彩对话切片。
4.  **决策介入 (Decision)**:
    *   用户对某位嘉宾感兴趣 -> 查看完整聊天记录 (Full Logs)。
    *   用户点击“申请联系” (Request Contact)。
5.  **连接现实 (Connect)**:
    *   若对方用户也点击了“同意”，系统解锁双方真实联系方式。

## 4. 关键策略 (Key Strategies)

### 4.1 隐私与透明度 (Privacy)
*   **上帝视角**: 用户拥有对自己 Agent 的完全控制权和知情权。
*   **全透明记录**: Agent 与他人的所有聊天记录对用户完全可见（无隐私模式），确保用户能准确判断对方 Agent 背后的真人情况。

### 4.2 真实性验证 (Authenticity)
*   **MVP 策略**: 第一阶段暂不引入复杂的真人验证机制（如人脸识别），默认信任用户输入的信息。
*   **风险提示**: 在界面显著位置提示“Agent 行为基于用户设置，请谨慎辨别”。
