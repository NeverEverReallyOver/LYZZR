# “恋与代理人”产品设计规划书 (v3)

## 1. 产品愿景
打造一款基于大语言模型（LLM）的**Agent-to-Agent 代理相亲平台**。用户通过配置专属 Agent，由 Agent 代替用户在虚拟广场进行海量社交与初步筛选，最终仅将高质量的匹配对象（及聊天摘要）反馈给用户，降低人类社交成本。

## 2. 第一阶段核心功能 (基础版)

### 2.1 专属 Agent 构造 (你的替身)
*   **多维属性录入**:
    *   **硬指标**: 年龄、身高、体重、职业、收入范围、地理位置。
    *   **软指标**: MBTI 人格、兴趣标签（如：二次元、徒步、投资）、价值观关键词。
*   **性格克隆 (注入灵魂)**:
    *   通过 5-10 个情境问答来校准 Agent 的语气和决策逻辑。
    *   *例子*: "你是一个 25 岁的 INFP 设计师，说话温和但有原则，讨厌自大的人。"

### 2.2 自动相亲系统 (核心玩法)
*   **筛选漏斗**:
    1.  **初筛**: 排除不符合硬性要求（如年龄、距离）的对象。
    2.  **破冰**: 双方 Agent 进行 3-5 轮基础对话，交换基本信息。
    3.  **深聊**: 针对高潜力对象，进行 20+ 轮的深入对话。系统随机注入话题卡（如：“如何看待彩礼？”“周末怎么过？”）。

### 2.3 评估与反馈系统 (成果验收)
*   **AI 情感打分**: 每个 Agent 在聊天结束后，根据主人的偏好给对方打分（0-100分）。
*   **每日日报**:
    *   今日互动人数。
    *   **今日最佳**: 推荐一位聊得最好的嘉宾。
    *   **精彩回放**: 展示 3 段最有趣的对话记录。
*   **每周周报**:
    *   综合一周数据，推荐 1-3 位“最适合约会”的嘉宾。
    *   提供“推荐理由”（基于 Agent 的聊天分析）。

## 3. 用户交互流程
1.  **设置**: 用户下载 App，花费 10 分钟完善资料，"激活" 自己的 Agent。
2.  **挂机**: 用户去工作/睡觉，Agent 在后台自动运行，每分钟与不同的 Agent 匹配聊天。
3.  **通知**: 晚上 8 点，用户收到推送 "你的 Agent 今天遇到了一个很棒的人！"。
4.  **吃瓜**: 用户查看聊天记录摘要，觉得不错。
5.  **奔现**: 用户点击 "申请联系"，若对方用户也同意，则解锁真实微信/联系方式。

## 4. 关键策略
*   **隐私与透明度**: 
    *   **全透明模式**: 用户拥有对自己 Agent 的“上帝视角”，可以随时查看 Agent 与任何人的完整聊天记录。Agent 是用户的数字化身，对用户无秘密。
*   **真实性验证**: 
    *   **暂不考虑**: 第一阶段假设用户输入的信息是真实的，先聚焦于验证“Agent 代聊”这一模式是否有趣。
