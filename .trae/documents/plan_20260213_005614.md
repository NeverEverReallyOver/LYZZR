# V2 版本开发计划：自动化与深度匹配升级

我们将针对您提出的三大痛点（安全、搜索、自动化），并结合用户体验优化，进行全面的 V2 版本升级。

## 📅 核心目标
从“单点手动测试”升级为“批量自动化匹配平台”，增强账号安全性，并提供直观的匹配结果排行。

## 🛠️ 详细实施计划

### 1. 账号安全体系升级 (Authentication)
**目标**：解决“裸奔”注册问题，确保用户账号安全。
*   **数据库变更**：
    *   `users` 表新增 `password_hash` 字段。
    *   (可选) 增加 `salt` 字段增强安全性。
*   **后端逻辑**：
    *   **注册**：引入 `bcrypt` 库，对用户输入的密码进行哈希加密后存入数据库。
    *   **登录**：校验输入的密码哈希值是否匹配。
    *   **迁移**：为现有 V1 老用户设置默认密码（如 `123456`）或提示重置。
*   **前端交互**：
    *   注册/登录页增加“密码”输入框（`type="password"`）。

### 2. 精准搜索与定向匹配 (Search & Target)
**目标**：解决无法找到特定好友或心仪对象的问题。
*   **功能实现**：
    *   **搜索栏**：在“嘉宾广场”顶部增加搜索框，支持按 `用户名(ID)` 或 `昵称` 模糊搜索。
    *   **定向挑战**：搜索结果卡片上增加“发起对话”按钮，点击后直接将该用户加入“待聊列表”。

### 3. 批量自动化相亲系统 (Batch Auto-Match)
**目标**：解决“点一下动一下”的繁琐操作，实现高效海选。
*   **核心逻辑重构**：
    *   **多选机制**：广场卡片支持“勾选”（Checkbox），允许用户一次选择 3-5 位嘉宾。
    *   **后台任务队列**：点击“开始批量接触”后，后端启动一个循环或并行线程。
    *   **自动对话流**：
        *   不再依赖 Streamlit 的手动刷新。
        *   代码自动执行：`打招呼` -> `对方回复` -> `我方追问` -> ... (设定每组对话上限，如 5 轮)。
        *   **并发处理**：尝试使用 `asyncio` 或线程池，让你的 Agent 同时跟 5 个人聊（海王模式！）。

### 4. 匹配排行榜与历史回溯 (Rank & History)
**目标**：结果可视化，让用户通过数据看懂谁最合适。
*   **数据持久化**：
    *   新增 `match_records` 表：记录 `user_a`, `user_b`, `chat_log`, `match_score`, `summary`。
*   **排行榜 UI**：
    *   **“真爱榜单”**：在侧边栏或独立 Tab 展示历史匹配过的人，按分数从高到低排序。
    *   **详情页**：点击榜单上的名字，弹窗展示当时的**完整聊天记录**和**裁判评价**。

### 5. (建议) 体验优化
*   **状态反馈**：批量聊天时，显示实时进度条（例如：`正在与 Alex 聊天 (3/5轮)...`）。
*   **防重复机制**：已聊过的用户，在广场上标记为“已聊”，避免重复无效社交。

---

## 🚀 执行步骤

1.  **数据库升级**：修改 `schema.sql`，添加密码字段和匹配记录表。
2.  **后端改造**：
    *   实现密码加密/校验。
    *   封装“自动聊天控制器”，支持输入两个 Agent，自动跑完 N 轮对话并返回结果。
3.  **前端重构**：
    *   改造登录页。
    *   改造广场（支持搜索、多选）。
    *   新增“排行榜”页面。
4.  **联调测试**：验证批量聊天是否稳定，分数是否准确写入数据库。
