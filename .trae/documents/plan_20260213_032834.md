The user points out a valid issue: after completing a single chat session (1-on-1 interaction), the results (chat history and evaluation report) are not being saved to the database, nor is the sidebar leaderboard updated to reflect this new interaction.

**Analysis of `app.py`:**
1.  **Batch Processing**: In the batch processing block (around line 325), `storage.save_match_record` is explicitly called to save the results.
2.  **Single Chat (Chat Room)**: In the single chat block (around line 376), after the conversation ends and the evaluation report is generated:
    *   It sets `st.session_state.report = report`.
    *   It calls `st.rerun()`.
    *   **CRITICAL MISSING STEP**: It does **NOT** call `storage.save_match_record`.
    *   This means the chat log and score are only stored in transient session state and lost upon refresh or new session, and never make it to the database/leaderboard.

**Plan:**

    *   Pass the correct arguments: `current_user.user_id`, `agent_b.user_id`, `session.history`, `total_score`, and `final_verdict`.
    *   This ensures single-chat interactions are treated exactly like batch interactions: persisted to DB and thus appearing in the leaderboard.
