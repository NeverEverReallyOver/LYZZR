The user wants to move two specific buttons ("Start Batch Auto-Match" and "Recommend One") to a specific location: **above the "Search Candidate" input box** in the "Candidate Pool" section, and they want them to be **in a single row**.

**Analysis of** **`app.py`:**

* Currently, the "Search" box is around line 166.

* The "Candidate Pool" grid is inside a form (`batch_match_form`).

* The "Start Batch Auto-Match" button is the `form_submit_button` *inside* the form, usually at the bottom.

* The "Recommend One" button is *outside* the form, in a separate column layout below the grid.

**Constraints & Challenges:**

1. **Form Submission**: The "Start Batch" button is a `form_submit_button`. If we move it *above* the form (the grid), it might still work as a submit button if it's inside the `with st.form:` block. Streamlit forms usually have the submit button at the bottom, but it's technically possible to place it anywhere *inside* the `with` block. However, putting it *above* the search box (which is outside the form?) might be tricky.

2. **Search Box Location**: The search box (`search_query`) is currently *outside* the form, *before* the grid loop.

3. **Layout**: To put them in "one row" above the search box, we need columns.

3. **Search Box**: `st.text_input(...)`

4. **Candidate Grid (Form)**: The grid of checkboxes.

**Problem**: The "Batch Match" button *needs* to submit the form that contains the checkboxes. If the button is *outside* the form (above the search box), it cannot be a `form_submit_button` for the grid below.

* **Solution A**: Move the entire form to wrap *everything* (buttons + search + grid). But search usually triggers a rerun on change, which forms prevent.
